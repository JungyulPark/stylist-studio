## Phase 1: 전환율 최적화 (이번 주)

> 목표: 무료 체험 → 유료 전환율 10%+ 달성

### 1-1. 결과 이미지 워터마크
- [ ] 생성된 헤어 이미지에 `kstylist.cc` 워터마크 자동 삽입
- [ ] Canvas API로 클라이언트에서 합성 (서버 비용 0)
- [ ] 공유할 때 자연스러운 브랜딩 효과
- **파일**: `src/App.tsx` (이미지 렌더링 부분)

### 1-2. 소셜 공유 OG 이미지 동적 생성
- [ ] 공유 시 결과 이미지가 OG 미리보기로 표시되도록
- [ ] Cloudflare Workers에서 동적 OG 이미지 생성 엔드포인트
- [ ] `og:image` 동적 URL: `/api/og?style=elegant&result=...`

### 1-3. 결과 페이지 퍼널 개선
- [ ] Upsell 클릭 시 즉시 결제 (현재: hair-selection으로 이동 → 다시 선택)
- [ ] "Hair Again" 클릭 → 결제 → 바로 새 스타일 생성 (스텝 줄이기)
- [ ] 타이머: "이 가격은 24시간만 유효합니다" 긴급성 추가

### 1-4. 이메일 수집
- [ ] 무료 체험 전 이메일 입력 (선택사항)
- [ ] 결과 이메일 발송 기능 활용
- [ ] 리마케팅: 7일 후 "새 스타일 트렌드" 이메일

## Phase 2: 바이럴 루프 강화 (다음 주)

> 목표: 오가닉 트래픽 30%+ 증가

### 2-1. 레퍼럴 시스템
- [ ] 추천 코드 생성: `kstylist.cc/?ref=ABC123`
- [ ] 추천인: 다음 분석 50% 할인
- [ ] 피추천인: 추가 무료 체험 1회
- **구현**: URL 파라미터 → localStorage → 결제 시 할인 적용

### 2-2. Before/After 갤러리 (소셜 프루프)
- [ ] 랜딩 페이지에 익명화된 Before/After 예시 슬라이더
- [ ] 사용자 동의 하에 결과 갤러리 공유
- [ ] "1,234명이 이미 체험했습니다" 카운터

### 2-3. 인스타그램 스토리 템플릿
- [ ] 결과를 9:16 비율로 자동 포맷
- [ ] "AI가 추천한 내 헤어스타일" 텍스트 + kstylist.cc 링크
- [ ] 1-tap 인스타 공유

## Phase 3: 수익 다각화 (2주 후)

> 목표: ARPU (객단가) $4.99 → $8+

### 3-1. 가격 체계 리팩토링
| 상품 | 현재 가격 | 목표 가격 | 비고 |
|------|-----------|-----------|------|
| 헤어 분석 1회 | $2.99 | $2.99 | 유지 |
| 풀 패키지 (헤어+패션) | $4.99 | $6.99 | 가치 인식 후 인상 |
| 월간 구독 (무제한) | 없음 | $9.99/월 | 신규 |
| 커플 패키지 | 없음 | $7.99 | 신규 |

### 3-2. 구독 모델 (Subscription)
- [ ] Stripe 구독 연동 (또는 Toss 정기결제)
- [ ] 월 $9.99 → 무제한 헤어/패션 분석
- [ ] "이번 달 3회 이상 사용하면 구독이 유리합니다" 넛지

### 3-3. 미용실 파트너십 (B2B)
- [ ] 미용실 전용 대시보드
- [ ] 고객이 결과 이미지를 미용실에 공유 → 미용실 예약 연결
- [ ] 미용실당 월 $29.99 구독 (고객 무제한 분석)

### 3-4. 광고 없는 프리미엄
- [ ] 무료 유저에게 결과 하단 배너 광고 (AdSense)
- [ ] 유료 유저는 광고 없음 + 고해상도 이미지

## Phase 4: 기술 개선 (3주 후)

> 목표: 속도, 안정성, 확장성

### 4-1. 성능 최적화
- [ ] 코드 스플리팅: `React.lazy()` + dynamic import
  - 현재 527KB 단일 JS → 라우트별 청크 분리
- [ ] 이미지 최적화: WebP 자동 변환 + lazy loading
- [ ] Lighthouse 점수 90+ 목표

### 4-2. SSR/Pre-rendering (SEO 핵심)
- [ ] SPA → SSG (Static Site Generation) 전환 검토
- [ ] Cloudflare Workers로 서버사이드 렌더링
- [ ] 또는: react-snap, prerender.io 활용
- **효과**: 네이버/구글 크롤러가 JS 실행 없이 콘텐츠 인덱싱

### 4-3. 악용 방지 강화
- [ ] 현재: localStorage만 (시크릿 모드 우회 가능)
- [ ] 추가: 브라우저 fingerprint (fingerprintjs)
- [ ] 추가: IP 기반 rate limiting (Cloudflare)
- [ ] 추가: Supabase 로그인 필수화 (무료 체험도)

### 4-4. 에러 모니터링
- [ ] Sentry 연동 (프론트엔드 에러 추적)
- [ ] API 응답 시간 대시보드
- [ ] 환불률 모니터링 알림
